Fastlane自动化打包部署

June 13, 2018   

我们的维语电商项目是 swift 和 OC 混编，所以每次手动打包发布测试版时巨慢，最近刚好把项目里的部分网络库从原来的OC版换到了swift版，正在慢慢的去 Objective-C 化。

久闻 fastlane 是自动化部署利器，今天就着实看了一下教程。简单把我踩过的坑记录下。

安装

sudo gem install fastlane -n /usr/local/bin

这里网上大部分的教程是说 sudo gem install fastlane 但至少我在 macOS High Sierra 上是不成功的，会报权限问题

ERROR: While executing gem … (Gem::FilePermissionError) You don’t have write permissions for the /usr/bin

参考链接Installing gem or updating RubyGems fails with permissions error，所以这里我选选择了指定目录安装。

然后进入项目工程中执行 fastlane init 它会询问一下配置选项，先后依次输入 AppID, AppleID 、密码、iTunes Connect Team 等完成基础配置后，会默认执行 $ bundle update，这里我等了差不多半小时，没任何反应😂 。终止后重新手动执行了下 $ bundle update 说没有权限，果断sudo 后很快就安装好了。

测试版需要上传到 fir.im 所以还需安装两个需要的插件 fastlane add_plugin versioning、fastlane add_plugin firim ，firim这个插件安装之前有依赖 firim

所以同理也要指定目录安装

$ sudo gem install -n /usr/local/bin firim -v '0.2.1' --source 'https://rubygems.org'

再安装 firim 插件就正常了。

配置

编辑 Fastfile

lane :to_firim do
  # 如果你用 pod install
  cocoapods
  # 如果你没有申请adhoc证书，sigh会自动帮你申请，并且添加到Xcode里
  #sigh(adhoc: true)
  # 以下两个action来自fastlane-plugin-versioning，
  # 第一个递增 Build，第二个设定Version。
  # 如果你有多个target，就必须指定target的值，否则它会直接找找到的第一个plist修改
  # 在这里我建议每一个打的包的Build都要不一样，这样crash了拿到日志，可以对应到ipa上
  increment_build_number_in_plist(target: "XXXXX")
  increment_version_number_in_plist(
    target: "XXXXX",
    version_number: '4.2.0'
    )
  # gym用来编译ipa
  gym(
    output_directory: './firim',
    export_options: {
      method: "ad-hoc", # 指定打包方式
      teamID: "XXXXX",
      thinning: "<none>"
    },
    scheme: "XXXXX"
    )
  # 上传ipa到fir.im服务器，在fir.im获取firim_api_token
  firim(firim_api_token: "XXXXXXXXXXXXXXXXXXXX")
end

将 target 、版本号、firim_token、TeamID等信息配置好后就可以 run fastlane to_firim 上传到 fir.im 了。

打包

第一次执行到 cocoapods 时报错了，查了下需要在 Pluginfile 文件中加上

gem 'cocoapods'

然后在执行，发现上传firim 时报错，一查发现没有设置token，去fir.im上生成了api_token后填写上后再执行终于成功了。

上fir.im 检查下，有的。perfect！

至此就初步完成最基本的测试版上传的配置，后续还需要再研究下去除一些不必要的log和警告，因为目前在打包时控制台会将所有的警告也输出。
