博客从hexo迁移到hugo

June 18, 2018   

利用端午假期，将自己的博客修改了一下。

之前用的是 hexo 框架做支撑，使用的 apollo 主题，很喜欢这款简洁又清新明快的主题。fork 过来后花了几天时间想自定义下，但 hexo 的主题开发文档实在是太少了，搜了下网上的博客也基本都千篇一律，没有太多实质性的知识点。在寻找的过程中发现了hugo 这个博客框架，这是一款用 go 写的，特点就是生成静态 HTML 非常快，无论是编译还是打包生成，个人使用下来体验就是比 Node.js 快很多，这也是 go 语言的特点。

非常喜欢 apollo 的风格，于是就自己撸了一个 apollo 风格的 hugo 主题叫 canary ，目前自己用的就是，初步完成了主页、博客列表、文章详情页等，其他关于我等其他页面后续还需要开发 [等我有灵感了再说]。这里只是记录一下开发过程中遇到的一些问题。

有关hugo的配置，这里就不在列举了，官网的Quick Start 写的也很清楚，配置可以说很简单。这大概也是 hugo 比 hexo 稍晚一点推出，目前 github 的 stars 数已经超过 hexo，是目前排名第二的静态博客框架。开发 canary 时也借鉴了 hugo-sustain 。

  • 主题中用到的图片资源是要放在 static/img下的，之前放在在 canary/images 下 本地server时找不到图片路径；
  • hugo 的 Summary 默认是70个字，对于中文有点太长，所以可以自己在文章中 使用 <!--more--> 进行切割，参看 Content Summaries；
  • $ hugo 会在与 content目录同级下生成 public/文件夹， 里面就是生成的静态页面。利用 github pages 搭建个人博客。
